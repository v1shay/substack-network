# Design decisions log — 2026-02-28

Record of design and workflow decisions made today.

## Work done

### Unfailed publications
- **retry_failed.py** — Retries queue rows with `status = 'failed'`. On success: inserts into `publications` and `recommendations`, enqueues new targets at depth+1, sets queue to `'crawled'`, and records the domain in the **unfailed** table (created by this script). Run after stopping the main crawler to avoid DB contention.
- **unfailed table** — `(domain, unfailed_at)`. Created by retry_failed.py; documents which domains were recovered from failure. Described in docs/database.md.
- **extract_failed.py** — Now also writes **data/unfailed_publications.html** from the unfailed table (plus existing failed_publications.html/csv). Run after retries to refresh both lists.
- **index.html** — Added “Unfailed publications” link (and in add_publication_lists.py); unfailed_publications.html included in gh_pages deploy.
- **Flow** — Retry → extract → investigate. Documented in private README (retry_failed, extract_failed, investigate_failed order and what each does).
- **merge-from-public.sh** (private) — Excludes merge-from-private.sh from being brought in from public when merging into private.

### PageRank distribution and power-law chart
- **pagerank_distribution.py** — Loads graph from cartographer.db, computes PageRank, sorts publications by rank. Fits power law PR(r) ≈ c·r^−α in log–log space; outputs JSON and/or an HTML fragment. Reports α, c, R²; caption notes Zipf-like when α ≈ 1.
- **Integration** — add_publication_lists.py runs the script with `--html-fragment` and embeds the chart **above** the “All publications in the database” table (db-publications.html). Chart: rank (x) vs PageRank (y, log scale).
- **Data sampling** — Blue points: first 1000 ranks + every 100th after that, so the tail is visible without overplotting. Red fit: smooth curve from 251 sampled points.
- **Styling** — Blue (data) drawn first, red (power-law fit) on top with transparency so both visible; red line wider; data points black, opaque. Tooltip: custom external div, fixed offset top-right of cursor so it doesn’t cover the anchor.
- **Caption** — Explains rank, α, c, R² and “Blue = data (first 1000 + every 100th thereafter); red = fitted curve.”
