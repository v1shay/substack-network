# Design decisions log — 2026-02-13

Record of design and workflow decisions made today.

---

## Summary of changes

1. **Script layout and paths** — Scripts live under `scripts/milestone01/`: `crawl.py`, `view_db.py`, `db_to_csv.py`, `summarize_db.py`, `centrality.py`, `visualize.py`. `gh_pages.py` stays at `scripts/`. Database and `data/` are at **repo root** (not under `scripts/`). README and all docstrings/error messages updated to use `scripts/milestone01/...`.

2. **CARTOGRAPHER_ROOT** — All scripts that need the repo root use `Path(os.environ.get("CARTOGRAPHER_ROOT", Path.cwd())).resolve()`. Default DB path: `root / "cartographer.db"`; outputs (e.g. `data/`, `index.html`) are under `root`. README documents the env var for CI or non–repo-root runs.

3. **setup.sh as single setup** — One entry point: `source setup.sh` creates venv, installs `substack_api` (editable) from sibling repo, and installs `requirements.txt` (networkx, numpy, scipy, pyvis). API repo path: default `../substack_api`, override with `SUBSTACK_API_REPO`. Removed redundant “pip install -r requirements.txt” from README; Setup section states that setup.sh installs all deps.

4. **Removed activate.sh** — Deleted `activate.sh`. In other terminals users run `source venv/bin/activate`. README, setup.sh success message, and `docs/multiple-repo-workflow.md` updated accordingly.

5. **Docs paths and links** — All docs updated to current layout:
   - **docs/visualization.md:** `milestone02-visualize.py` / `milestone02-centrality.py` → `scripts/milestone01/visualize.py`, `scripts/milestone01/centrality.py`.
   - **docs/database.md:** Script → `scripts/milestone01/crawl.py`; DB at repo root; inspection uses `python scripts/milestone01/view_db.py`; removed `cd scripts`; sqlite3 and Python examples use `cartographer.db` at repo root.
   - **docs/data-analysis.md:** Implementation reference → `scripts/milestone01/centrality.py`.
   - **docs/development-plan.md:** All crawler/centrality paths → `scripts/milestone01/crawl.py`, `scripts/milestone01/centrality.py`; project structure tree updated (cartographer.db at root, `scripts/milestone01/` with crawl, view_db, centrality, visualize).
   - **docs/README.md:** Quick Start “Working” line → `scripts/milestone01/crawl.py`, cartographer.db at repo root.

6. **Code fixes** — `centrality.py` and `visualize.py`: added `import os` so `os.environ.get("CARTOGRAPHER_ROOT", ...)` works. `requirements.txt` comment updated to reference `scripts/milestone01/` and setup.sh.

## Decisions

- **Run from repo root:** Scripts assume cwd is repo root (or `CARTOGRAPHER_ROOT` is set). DB and `data/` at repo root keep paths simple and match README.
- **Setup only via setup.sh:** No separate activate script; no manual `pip install -r requirements.txt` in normal workflow.
- **API repo configurable:** `setup.sh` uses sibling `../substack_api` by default; `SUBSTACK_API_REPO` overrides for different layouts or CI.
